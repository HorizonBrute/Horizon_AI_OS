# Handoffs — Horizon AIOS

This directory stores session handoff documents written by the `/handoff` skill. A handoff captures session state so a future session (or human reviewer) can orient and continue without reconstructing context.

## Filename convention

`YYYY-MM-DD_HHMMSS_<project-name>.md`

Generated by `/handoff` — do not rename manually.

## Lifecycle and Pruning

Handoff documents are temporary by design. They are living state snapshots, not permanent records. The authoritative history of what was done lives in git commits and objectives files.

**Automated pruning:** `horizon_aios_maintain_logs.py` prunes this directory by **total size budget**. The default threshold is `AIOS_HANDOFFS_MAX_SIZE_MB = 500 MB` (configurable in `$HORIZON_ETC/aios_local.conf`). When the directory exceeds the budget, the oldest files are deleted first until it is back under budget. Set to `0` to disable handoff pruning.

**What to keep:** If a handoff represents a milestone you may want to reference later, move it elsewhere or extract the relevant decisions into the linked objective file before the pruning window closes.

**What to discard:** Handoffs from completed work with no open deferred items have no long-term value — pruning them is appropriate.

## Reading a handoff

Paste the handoff filename (or full path) into a new session prompt. The agent at the top of the file will orient itself and begin from the "Next Session Entry Point" section.
